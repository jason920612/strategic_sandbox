# RFC-050：事件、隱藏真相與情報失真

- Status: Draft
- Type: GDD / Simulation Design
- Scope: 事件系統
- Version: 0.1

## 1. 事件原則

事件不是固定劇本，而是由世界狀態、權重與事件鏈生成。

## 2. 事件生成

事件由條件與權重決定。例如：

```text
如果穩定低 + 工人不滿 + 失業高 + 媒體自由高
則「全國罷工」權重上升。
```

## 3. 事件鏈

事件鏈是條件連鎖，不強制歷史。

例：

```text
工人罷工
→ 政府鎮壓
→ 媒體曝光
→ 學生抗議
→ 軍方要求戒嚴
→ 國會調查
→ 政變風險上升
```

每一步都可以被制度、派系、玩家決策、情報能力與隨機權重改變。

## 4. 事件文字風格

依事件來源使用不同格式：

- 報紙標題
- 內閣報告
- 情報簡報
- 角色對話
- 系統訊息

## 5. 隱藏真相

同一事件可以有多層：

```text
真相：
地方利益集團煽動罷工阻止中央稅務改革。

政府報告：
工人因薪資不滿自發罷工。

媒體敘事：
新稅法壓迫地方民生，引發抗議。
```

## 6. 資訊準確度

玩家收到的事件敘述受以下影響：

- 情報能力
- 官僚忠誠
- 媒體自由
- 地方控制
- 司法獨立
- 腐敗
- 派系滲透
- 玩家是否被架空

## 7. 事件資料欄位

事件應支援：

- event_id
- category
- trigger_conditions
- weight_modifiers
- visible_report
- true_cause
- possible_options
- effects
- followup_events
- source_type
- information_accuracy_required
- tags

## 8. v0.1 事件

v0.1 先做少量事件，重點是事件引擎：

- 罷工
- 學生抗議
- 媒體醜聞
- 地方抗稅
- 軍方施壓
- 情報擴權
- 經濟放緩
- 預算危機
- 政變傳聞
- 派系妥協

# RFC-010：第一可玩原型 v0.1

- Status: Draft
- Type: GDD / TDD / Roadmap
- Scope: v0.1 原型
- Version: 0.1

## 1. v0.1 目標

v0.1 的目標不是做出完整遊戲，而是驗證以下問題：

1. 玩家政策是否能影響派系與穩定？
2. AI 國家是否能自然演化出不同局勢？
3. 事件系統是否能產生有趣危機？
4. 國家是否會自然走向崩潰、政變或內戰風險？
5. GDP、稅收、預算是否能支撐內政玩法？
6. SVG 地圖與 UI 面板是否足夠支援除錯與早期可玩性？

## 2. v0.1 必做功能

### 2.1 玩家操作

- 玩家可選一個國家
- 玩家可修改預算
- 玩家可啟用政策
- 玩家可查看派系、穩定度、GDP、稅收
- 玩家可暫停、繼續、調整速度

### 2.2 世界

- 1930 起始
- 20–30 個主要國家
- 小國可先作為空殼
- 所有國家有 GDP、稅收、穩定度、軍力、派系簡表
- AI 國家可自動選政策

### 2.3 地圖

- 2D SVG 地圖
- 可先用省份節點圖
- 點擊國家或省份顯示面板
- 國家顏色代表控制者
- 事件 log 顯示在 UI

### 2.4 經濟

只做：

- GDP
- 稅收
- 預算

簡化預算分類：

- 行政
- 軍事
- 教育
- 福利
- 情報
- 基建
- 產業

### 2.5 派系

第一版建議 6–8 個派系：

- 軍方
- 官僚
- 工人／工會
- 地方勢力
- 媒體
- 情報部門
- 學生
- 技術菁英

每個派系只需：

- 支持度
- 影響力
- 激進度
- 滿意度變化原因 log

### 2.6 政策

每個領域 3–5 個政策：

- 稅制
- 軍費
- 教育
- 媒體
- 情報
- 福利
- 行政改革

### 2.7 事件

早期只做少量事件，重點是事件引擎：

- 罷工
- 學生抗議
- 軍方施壓
- 媒體醜聞
- 地方抗稅
- 情報部門要求擴權
- 經濟放緩
- 預算赤字
- 派系妥協
- 政變傳聞

## 3. v0.1 不做

- 完整人物系統
- 完整國會與司法
- 完整戰爭
- 完整外交會議
- 完整產業鏈
- 通膨、債務、貨幣
- 核武
- 完整內戰
- 真實多邊形地圖

## 4. v0.1 可玩目標

玩家選擇一個國家，在 SVG 地圖與面板 UI 中：

1. 1930 開局
2. 調整預算
3. 選擇政策
4. 觀察派系與穩定變化
5. 觀察 AI 國家自動演化
6. 看到事件與危機
7. 可跑到 2000 年
8. 可匯出 log 與 CSV

## 5. 驗收標準

- 同 seed 結果可重現
- 可連續跑 70 年不崩潰
- 至少 20 國自動運作
- 至少 20 個政策
- 至少 10 個事件模板
- 至少 6 個派系
- SVG 地圖可更新國家顏色
- 可輸出年度統計 CSV
